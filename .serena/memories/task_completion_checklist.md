# Чек-лист по завершению задачи

После завершения кодовых изменений рекомендуется:

1. **Frontend:** Выполнить линтинг: `cd frontend && npm run lint`. Исправить предупреждения и ошибки ESLint.
2. **Сборка фронта:** Убедиться, что сборка проходит: `cd frontend && npm run build`.
3. **PHP:** Проверить синтаксис/логику вручную или запуском затрагиваемых скриптов/эндпоинтов; при наличии phpstan/phpcs — запустить анализ для изменённых путей.
4. **Тесты:** Если в задаче предусмотрены тесты — запустить их; для вебхуков и скриптов см. `src/scripts/README_TESTS.md`, `doc/TESTING_WEBHOOKS.md`.
5. **Документация:** При изменении API, переменных окружения или сценариев деплоя — обновить соответствующие файлы в `doc/` или README.
6. **Память/таски:** При значимых архитектурных изменениях обновить память проекта (Serena) или статус в `tasks/`, если задача относится к таскам (например google-sheets-integration).
